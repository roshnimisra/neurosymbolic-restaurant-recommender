# Neurosymbolic Restaurant Recommender: Philadelphia Case Study

This repository contains a comparative research study and implementation of a **Neurosymbolic Retrieval Pipeline** for local business discovery. Utilizing the Yelp Open Dataset, this project evaluates the effectiveness of three distinct retrieval architectures: **Symbolic Logic**, **Neural Embeddings**, and a **Hybrid "Filter-then-Rank"** system.

## Project Overview

Traditional recommendation engines often force a trade-off between **hard constraints** (e.g., "Is it open at 10 PM?") and **soft semantics** (e.g., "Does it have a cozy date-night vibe?"). This project implements a dual-stream architecture to bridge this gap:

1.  **The Symbolic Baseline:** A high-precision filtering engine using structured operational data (39+ binary/numeric attributes).
2.  **The Neural Baseline:** A semantic search engine utilizing LLM-synthesized "Vibe Summaries" and vector embeddings.
3.  **The Hybrid Model:** An integrated pipeline that utilizes symbolic logic for hard-pruning and neural similarity for final ranking.

## Data Engineering Pipeline

The project focuses on a curated subset of **2,800 restaurants** in Philadelphia, PA. The data was subjected to a multi-stage feature engineering pipeline:

### **1. Hierarchical Feature Representation**
Categories were refactored into a hierarchical schema to improve NLU (Natural Language Understanding) accuracy:
* `cuisine_group`: Broad regional identifiers (e.g., East Asian).
* `cuisine`: Specific national styles (e.g., Japanese, Korean).
* `key_menu`: Signature items (e.g., Sushi, Ramen).
* `establishment_type`: Venue classification (e.g., Izakaya).

### **2. Structured Attribute Encoding**
* **Binary & Ordinal Filtering:** Flattened the raw `attributes` JSON into 39 discrete columns. This includes both prefixed groups (e.g., `ambience_*`, `meal_*`) and standalone operational tags (e.g., `WiFi`, `WheelchairAccessible`). Encoded as binary or ordinal values, these provide the **Symbolic Baseline** with high-precision constraints for logical filtering.

### **3. Temporal & Spatial Engineering**
* **Decimal Time Transformation:** Raw operating hours were converted to a linear decimal format to support numeric comparison logic ($Hour + \frac{Minutes}{60}$). Closing times exceeding midnight were handled using $24h+$ indexing (e.g., 2:00 AM becomes $26.0$) to maintain range-query integrity.
* **Geospatial Join:** Neighborhood labels (e.g., Fishtown, Rittenhouse) were engineered via a spatial join between restaurant coordinates and official municipal GeoJSON boundaries.

### **4. Semantic Synthesis**
A "Review Summary" column was generated by distilling the top 10 most "useful" user reviews along with category tags into a 150-word descriptive paragraph using an LLM (Gemini/GPT), providing the dense textual context required for the **Neural Baseline**.

## Getting Started

### **Prerequisites**
* Python 3.9+
* Pandas (with Nullable Integer support)
* GeoPandas & Shapely (for spatial joins)
* Sentence-Transformers (for Neural Baseline)

### **Installation**
1. Clone the repository:
   ```bash
   git clone https://github.com/your-username/neurosymbolic-restaurant-recommender.git
   ```
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
